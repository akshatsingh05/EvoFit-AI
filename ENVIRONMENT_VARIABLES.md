# Environment Variable Guide

Copy `backend/.env.example` → `backend/.env` and `frontend/.env.example` →
`frontend/.env`, then fill in real values. Never commit `.env` files.

## Backend (`backend/.env`)

| Variable | Required | Default | Notes |
|---|---|---|---|
| `ENVIRONMENT` | No | `development` | `development` \| `staging` \| `production`. Controls docs exposure, HSTS header, and secret validation. |
| `DEBUG` | No | `true` | Verbose logging + error detail in responses when `true`. Set `false` in production. |
| `DATABASE_URL` | Yes | `sqlite:///./evofit.db` | SQLite locally; `postgresql://USER:PASS@HOST:5432/DB` in production. |
| `DB_POOL_SIZE` | No | `5` | SQLAlchemy connection pool size (Postgres only). |
| `DB_MAX_OVERFLOW` | No | `10` | Extra connections allowed beyond pool size (Postgres only). |
| `JWT_SECRET_KEY` | **Yes in production** | dev placeholder | Signs access tokens. App refuses to start in production with the placeholder value. Generate with `python -c "import secrets; print(secrets.token_urlsafe(64))"`. |
| `SECRET_KEY` | **Yes in production** | dev placeholder | General app secret, kept separate from `JWT_SECRET_KEY` so each can rotate independently. Same generation method. |
| `JWT_ALGORITHM` | No | `HS256` | |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | No | `15` | Access token lifetime. Short by design — the frontend transparently refreshes via `/auth/refresh`. |
| `REFRESH_TOKEN_EXPIRE_DAYS` | No | `30` | Refresh token lifetime. Tokens are also single-use (rotated on every refresh) and revocable server-side (logout). |
| `CLAUDE_API_KEY` | No | empty | Reserved for a future `ClaudeProvider`. Not read by any code path today — the app runs entirely on the built-in rule-based engine. |
| `OPENAI_API_KEY` | No | empty | Reserved for a future `ChatGPTProvider`. Same as above. |
| `AI_PROVIDER` | No | `rule_based` | Reserved; only `rule_based` is implemented. |
| `CORS_ORIGINS` | Yes | `http://localhost:5173` | Comma-separated list. Must exactly match your deployed frontend origin(s). |
| `TRUSTED_HOSTS` | **Yes in production** | `*` | Comma-separated allowed `Host` header values, or `*`. Set to your real backend domain(s) in production. |
| `FRONTEND_URL` | No | `http://localhost:5173` | Informational; not currently used to build links, but available for future email/redirect features. |
| `BACKEND_URL` | No | `http://localhost:8000` | Informational, same as above. |
| `RATE_LIMIT_ENABLED` | No | `true` | Disables rate limiting entirely if `false` (e.g. for load testing). |
| `AUTH_RATE_LIMIT` | No | `10/minute` | Applied per-client-IP to `/auth/signup`, `/auth/login`, `/auth/forgot-password`. slowapi syntax: `<count>/<second\|minute\|hour\|day>`. |

## Frontend (`frontend/.env`)

| Variable | Required | Default | Notes |
|---|---|---|---|
| `VITE_API_BASE_URL` | Yes | `http://localhost:8000` | Backend API origin. **Baked in at build time** (Vite env vars are resolved during `npm run build`, not read at runtime) — set this correctly *before* building for each environment (see Docker `ARG VITE_API_BASE_URL` / Vercel project settings). |

## Generating secrets

```bash
python -c "import secrets; print(secrets.token_urlsafe(64))"
```

Run this twice to get distinct values for `JWT_SECRET_KEY` and `SECRET_KEY`.

## Where these get set in each deployment target

- **Local dev**: `.env` files (git-ignored).
- **Docker Compose**: `docker-compose.yml` environment blocks, or a root
  `.env` file Compose reads automatically. `VITE_API_BASE_URL` is a build
  `arg`, not a runtime `environment` var, for the frontend service.
- **Render**: `render.yaml` `envVars` — `sync: false` entries must be filled
  in manually in the Render dashboard after first deploy (e.g. `CORS_ORIGINS`
  once you know your Vercel URL). `JWT_SECRET_KEY`/`SECRET_KEY` use
  `generateValue: true` and are generated by Render automatically.
- **Vercel**: Project Settings → Environment Variables → `VITE_API_BASE_URL`
  set to your Render backend URL, for the Production/Preview environments
  you care about. Redeploy after changing it (build-time variable).
